Antigravity Hub — Mobile Web Optimization

GUI & CSS Analysis and Modular Override Layer for Desktop React Web App v2.3.1

Author: Antigravity Agent Target Version: 2.3.1 Date: Loading date...

🔍 Root Cause Diagnostics

1. Bottom Input Box Clipping

Mobile browsers (iOS Safari, Android Chrome) calculate 100vh based on the full screen height including address bars. When bars are visible, sticky.bottom-0 input containers get submerged under browser UI.

2. Top Header & Notch Clipping

Sticky header elements and breadcrumb bars lacked env(safe-area-inset-top) support. On modern mobile screens with notches or camera cutouts, controls collide with status bars.

3. Touch Target & Accessibility

Desktop UI buttons for sidebar toggles and window collapse icons used 28px/32px dimensions, which fall below mobile touch guidelines (min 38px–44px).

🏗️ Architectural Strategy: Modular Override

To avoid manual CSS merging whenever upstream Electron app updates are compiled into main.js or compiled_tailwind.css:

  • Zero modifications to core bundles: compiled_tailwind.css and main.js remain untouched.
  • Standalone CSS File: Created mobile-override.css linked in index.html directly after compiled styles.
  • Version Manifest: Created index.json specifying version 2.3.1 and extension entry points.

💻 Custom CSS Overrides Snippet

/* mobile-override.css (Antigravity Hub v2.3.1) */
@supports (height: 100dvh) {
  html, body, #root, .h-screen, .h-\[100vh\] {
    height: 100dvh !important;
    min-height: 100dvh !important;
  }
}

@media (max-width: 768px) {
  header, .sticky.top-0 {
    padding-top: env(safe-area-inset-top, 0px);
  }
  
  #antigravity\.agentSidePanelInputBox, .sticky.bottom-0 {
    padding-bottom: calc(env(safe-area-inset-bottom, 0px) + 8px) !important;
  }
  
  #antigravity\.agentSidePanelInputBox textarea {
    font-size: 16px !important; /* Prevents iOS Safari auto-zoom */
  }
}
